218  /*
219  * Type of the value in an iod can be either a single type that is
220  * always overwritten when updated, or it can be an array of EQUAL sized
221  * records where the record is updated atomically. Note that an akey can
222  * only support one type of value which is set on the first update. If
223  * user attempts to mix types in the same akey, the behavior is
224  * undefined, even after the object, key, or value is punched. If
225  * \a iod_type == DAOS_IOD_SINGLE, then iod_nr has to be 1, and
226  * \a iod_size would be the size of the single atomic value. The idx is
